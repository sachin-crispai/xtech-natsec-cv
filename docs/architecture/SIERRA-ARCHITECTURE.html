VEGAS TIER

Global Brain Centers  ·  2–3 sites

The apex of the hierarchy. MEAD and POWELL are H200 super-clusters (16 cards each, 2.2 TB VRAM per site) that train the foundation models powering the entire fleet. Model weights flow downstream to TAHOE-tier controllers via VPN fabric. mamba and other operator workstations also live in this tier but are not compute nodes — they are management interfaces that may be physically anywhere and always connect via VPN.

TAHOE TIER

Operational Hubs inside SIERRA  ·  ~200 nodes

SUB-CONTROLLERS

ALMANOR / DONNER Class  ·  4–5 per hub  ·  ~800–1,000 total

Named after small Sierran alpine lakes. These nodes sit between TAHOE hubs and the raw EDGE fleet. They perform CV inference pre-processing, aggregate sensor data, and serve as the first failover target when an EDGE node loses its primary. GPU class is L40S or RTX 6000 Ada — CV-optimized, not training-capable.

Page 3 of 3 — Glossary

Glossary

Key terms for the SIERRA control center architecture

TermDefinition
SIERRAThe physically and logically isolated secure zone in Las Vegas, NV. All operational compute lives inside SIERRA. Entry requires VPN authentication regardless of the physical location of the connecting node.
VEGAS TierThe global brain — 2–3 geographically distributed H200 training clusters outside SIERRA, connected via VPN. Trains foundation models and pushes weights downstream. Top of the hierarchy.
TAHOE TierOperational hub-level controllers inside SIERRA. ~200 H100 nodes. Coordinate missions, run tactical inference, manage the EDGE fleet. TAHOE is the central hub all sub-controllers sync through.
Sub-ControllerIntermediate layer (ALMANOR/DONNER class) between TAHOE hubs and EDGE nodes. Aggregates sensor data, runs CV inference pre-processing, and serves as first failover target for EDGE nodes. L40S / RTX 6000 Ada class — not training-capable.
EDGE Tier~100,000 drone master nodes at field periphery. Limited GPU (inference only). Each is assigned a primary and backup controller. Operates autonomously during connectivity gaps and fails over automatically.
mambaOperator workstation in the VEGAS tier. May be physically anywhere — inside the Vegas center, hotel, or field. Not a compute node. Always VPN regardless of location.
VPN FabricEncrypted tunnel linking VEGAS brain centers to SIERRA. Mutual certificate authentication. All cross-boundary traffic traverses this path.
FailoverWhen an EDGE node loses heartbeat with its primary controller: retry (N times) → promote backup controller → sync operational state → continue mission under backup custody.
Custody TransferHand-off of an EDGE node’s swarm assignment from one controller to another. Triggered automatically on failover, or manually by an operator when a primary controller recovers.
NDR / HDRInfiniBand network generations. NDR (Next Data Rate) = 400 Gb/s — used in VEGAS clusters. HDR (High Data Rate) = 200 Gb/s — used in SIERRA TAHOE clusters.
NVLink 4NVIDIA high-bandwidth GPU-to-GPU interconnect inside H200 nodes. Enables full memory sharing across all 16 GPUs in MEAD/POWELL configurations (~900 GB/s per direction).
PFLOPS FP8Peta (10¹&sup5;) floating-point operations per second at 8-bit precision. Standard AI training throughput metric. H200 SXM5 delivers ~3.96 PFLOPS FP8 per card.
Node namingWestern lakes convention. VEGAS tier: large Nevada/Arizona reservoirs (MEAD, POWELL, HAVASU). TAHOE tier: large California lakes (SHASTA, OROVILLE, BERRYESSA, FOLSOM). Sub-controllers: small Sierran alpine lakes (ALMANOR, DONNER, TENAYA, CASCADE, CONVICT, TOPAZ).
Autonomous floorMinimum operational capability an EDGE node maintains when fully disconnected from SIERRA — local inference on existing models, swarm cohesion, and event logging for later sync.
